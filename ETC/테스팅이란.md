### 무작위 고장
실행 중 죽어버리는 프로그램, 항상 다른 지점에서 죽는다. 단일 스레드이고, C의 표준 라이브러리만 쓴다. 어떤 오류가 있을까?

1. 랜덤 변수
	* 프로그램 내 난수 사용
	* 실행마다 다른 경로로 실행
	* 
2. 초기화하지 않은 변수
	* 언어에 따라 임의값 할당해버리기 때문

3. 메모리 누수
	* 프로그램 메모리가 부족한 경우

4. 외부 의존성
	* 다른 응용프로그램, 컴퓨터, 자원에 의존
	* 의존관계 많으면 죽을 수 있다.

### No 테스트 도구
테스트 도구를 사용하지 않고 웹 페이지에 부하 테스트를 실행하려면 어떻게 하면 되는가?

* 성능이 중요한 시나리오 찾기
* 성능 목표치를 충족 여부 알려주는 지표 확인
	* 반응 시간
	* 처리량
	* 자원 사용률
	* 시스템이 감내할 수 있는 최대 부하

* 테스트 도구 만들기
	* 수천 명의 가상 사용자
	* 수천 개 스레드를 갖는 다중 스레드

* 수집 데이터와 수용 가능 데이터와 비교

### 펜 테스트
펜을 어떻게 테스트하겠는가?

* 육하원칙에 따라 많은 질문 던져야 한다. 
* 테스트 하는 것을 정확히 이해하고 시작해야 한다.

1. 사용자의 특성, 펜의 용도를 파악한다.
2. 사용감, 무게감, 번짐 등 사용자를 고려하여 평가 기준을 명확하게 정의한다.
3. 테스트를 한다.
	- 사실 검사 : 재질, 색의 잉크 확인
	- 의도된 사용법 : 옷감 위에 잘 쓰이나?
	- 의도된 사용법 : 옷에 그린 그림이 잘 지워지는가?
	- 안정성 : 사용자(5-6세 유아)에게 무해한가?
	- 의도하지 않은 사용 : 밟거나 던지거나 등 감내하나?
(책 참고 P.555)

### ATM테스트
분산 은행 업무 시스템을 구성하는 ATM을 어떻게 테스트하겠는가?

1. 시스템 파악 : 업무 시스템의 설계 모형, 시간대별 사용자, 사용자 연령 등 파악
2. 테스트 도구 활용 or 만들기 

정상작동

- 시스템이 정상적으로 작동 하는가
- 트랜젝션이 잘 수행 되는가?
- 정상 작동 범위가 어느정도인가?

비정상작동
- 원인이 무엇인가?
- 서버가 둘 다 죽었을 때
- 서버 일부가 죽었을 때
- 어느 정도의 부하에 죽는가

1. 가정을 분명히 한다.
사용자, 용도, 주어지는 테스트 도구

2. 컴포넌트를 나눈다.
로그인/인출/입금/조회/이체

3. 수동 테스트 및 자동 테스트
수동 테스트는 모든 오류의 경우를 검사할 수 있어야한다.
자동 테스트는 표준적 시나리오들을 자동화 해야한다.
시스템에 관계된 우선순위를 이해. (이 예제의 경우 보안, 안전성 최우선)
